Este proyecto consiste en un Diagrama de Entidad-Relación (DER) para un sistema de base de datos. El DER es una herramienta esencial para diseñar, gestionar y comprender cómo se relacionan las diferentes entidades dentro de una base de datos. A continuación, se desglosan algunas entidades clave y sus atributos dentro de la base de datos:

  • Cursos: cod_curso, curso, modalidad, sección, jornada
  • Maestros: cod_maestro, nombre, apellidos, dirección, teléfono, email
  • Materias: cod_materia, materia, cod_maestro
  • Notas: rne, cod_curso, cod_materia, año, p1, p2, p3, p4, promedio, recuperación1, recuperación2
  • Ventas: nventas, cod_empleado, fecha_venta, rne
  • Detalle: nventa, cod_producto, precio_venta, cantidad, total, impuesto, totalapagar

  • Empleados: cod_empleado, nombre, apellidos, dirección, teléfono, email

  • Botones Interactivos: Permiten a los usuarios ingresar números y seleccionar operaciones matemáticas.
  • Pantalla: Muestra los números ingresados y los resultados de las operaciones.
  • Operaciones Matemáticas: Implementadas en Python para realizar cálculos precisos.
  • Interacción Cliente-Servidor: La lógica de backend está gestionada por Python, utilizando un marco como Flask o Django.
